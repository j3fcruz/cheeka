# 🌐 Overview

**Cheeka LAN Messenger** is a secure, offline-first communication tool designed for private messaging over local networks without requiring:

* Internet access
* External servers
* User accounts
* Cloud infrastructure

It enables instant encrypted communication between devices on the same LAN using peer-to-peer architecture and automatic network discovery.

Built for environments where privacy, isolation, and speed are critical.

---

# 🔐 Why Cheeka?

Most messaging platforms today rely on:

* Cloud servers
* External authentication
* Internet connectivity
* Data collection pipelines

Cheeka removes all of that.

Instead, it operates entirely within your local network:

✔ No data leaves your LAN
✔ No user tracking
✔ No external dependency
✔ No login required

Your network becomes your messaging system.

# ✨ Key Features

## 🔎 Zero-Config Device Discovery

Automatically detects active peers on the same LAN using broadcast discovery.

* No manual IP setup

# 🔒 Security Model

## Local-Only Communication

All communication stays inside the local network.

No internet routing is used.

---

## Encryption Flow

```text id="cheeka_sec"
Message
  │
  ▼
AES-256 Encryption
  │
  ▼
LAN Transmission (Sockets)
  │
  ▼
Decrypted on Peer Device
```

# ❓ Frequently Asked Questions

### Does Cheeka require internet?

No. It works entirely on local networks.

---

### Can messages be intercepted?

Messages are AES-256 encrypted before transmission, making interception unreadable.

---

### Do I need a server?

No. Cheeka is fully peer-to-peer.

---

### Can it work in offices or schools?

Yes, as long as devices share the same LAN.

---

### Does it store chat history?

By default, messages are ephemeral unless persistence is enabled.

# 🔧 Troubleshooting

## Devices Not Detected

Check:

* Same LAN/Wi-Fi network
* Firewall permissions
* UDP broadcast allowed

---

## Messages Not Sending

Possible causes:

* Peer disconnected
* Network restrictions
* Port blocking

---

## File Transfer Fails

Ensure:

* Sufficient disk space
* Stable LAN connection
* Same encryption handshake version
